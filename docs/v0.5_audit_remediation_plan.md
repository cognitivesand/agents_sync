# v0.5 Audit Remediation Plan

**Date:** 2026-05-21
**Branch:** `feat/v0.5-mcp-server`
**Input:** [`docs/global_audit_2026_05_21.md`](./global_audit_2026_05_21.md) + 10 per-slice JSON reports in [`docs/audit/`](./audit/)
**Already partially addressed by:** `86fe429` ("audit fix : 1,2,3 and 4") — slice 02 (mcp_secret_policy), slice 04 (key_field asymmetry + non-dict overwrite), and parts of slice 06 (discovery error paths)

This plan delivers the remaining audit fixes in **4 sequential phases**, each one PR / commit. Priorities are explicit in the order:

1. Antipattern elimination (severity-driven)
2. DRY
3. Hardcoding elimination

Phases are ordered so that earlier phases unblock later ones — e.g. polymorphic `FileLayout` (Phase 2) is a precondition for splitting `discovery.py` cleanly (Phase 4).

## Phase 1 — Critical correctness antipatterns

**Goal:** close every CRITICAL and MAJOR-correctness finding that puts user data at risk under NFR-01 ("never lose user content").
**Risk if skipped:** silent data loss, undetectable failure modes, downgraded contracts that the docstrings still advertise.

### Scope

| # | Audit ref | File | Change |

Use `fcntl.flock` (POSIX) / `msvcrt.locking` (Windows) gated by `sys.platform`, wrapped in a context manager. Retire the "lightweight guard" docstring downgrade introduced by `86fe429` — either honor the original "never partially overwritten" contract or remove it entirely.
