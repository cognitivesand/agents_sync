# agents_sync v0.3 - Implementation Plan

This document defines the engineering plan for `v0.3`, focused on first-class Windows support while preserving existing Linux behavior.

## Why v0.3

Current `v0.2` functionality is largely cross-platform in core sync logic, but the operational experience is Linux-only:

- Installation uses `install.sh`.
- Service supervision assumes `systemd --user`.
- Default paths and documentation are Linux-centric.

`v0.3` closes that gap by making install, runtime supervision, defaults, and validation truly cross-platform.

## Goals

- Add supported Windows installation and background execution.
- Keep Linux behavior stable and backward-compatible.
- Preserve functional invariants: bidirectional sync, archive-before-destructive-write, and loop suppression.
- Keep daemon resilience under Windows filesystem behavior (locking, rename timing, case-insensitive paths).

## Non-goals

- Replace polling with platform file watchers (`inotify`, `ReadDirectoryChangesW`, etc.).
- Add multi-host or cloud sync.
- Add GUI management.
- Drop Linux `systemd` flow.

## Baseline findings (from current code)

- Core modules (`sync.py`, `state.py`, `archive.py`) already use `pathlib` and mostly portable operations.
- Main platform constraints are installer/service UX and OS-specific defaults.
- A few runtime details need hardening for Windows:
  - text write newline consistency
  - transient `PermissionError`/sharing violations during replace/move
  - path/name edge cases on case-insensitive filesystems.

## v0.3 Workstreams

### 1) Platform and path abstraction

- Introduce explicit platform-aware default path helpers in `config.py`.
- Keep all existing CLI override flags unchanged.
- Preserve `~` expansion support for explicit user-provided paths.

Default strategy:

- Linux/macOS style defaults remain as-is.
- Windows defaults move to user profile conventions:
  - config under `%APPDATA%\\agents-sync\\`
  - state under `%LOCALAPPDATA%\\agents-sync\\state\\`
  - Claude/Codex source dirs remain user-home based unless explicitly overridden.

Deliverables:

- Refactored `DEFAULTS` construction in `config.py`.
- Unit tests for default resolution on Windows and Linux.

### 2) Windows installer and service supervision

- Keep `install.sh` for Linux.
- Add `install.ps1` and `uninstall.ps1` for Windows.

`install.ps1` responsibilities:

- Verify Python/`uv` availability (or provide clear guidance).
- Create/update local venv.
- Install/update `agents-sync` launcher.
- Seed config file if missing.
- Register a per-user Scheduled Task for daemon startup and restart behavior.

`uninstall.ps1` responsibilities:

- Remove scheduled task.
- Remove launcher.
- Leave user data/config by default (with optional explicit cleanup flag).

Windows supervision model:

- Primary: Task Scheduler (`schtasks`/ScheduledTasks module), trigger at user logon.
- Process command: `agents-sync --config <path>`.

Deliverables:

- New PowerShell installer scripts.
- README install/run sections for both Linux and Windows.

### 3) Runtime hardening for Windows filesystem semantics

- Add bounded retries around atomic file replace and directory swaps where transient locks are common.
- Ensure atomic write helpers enforce LF output bytes to avoid CRLF drift affecting digests and cross-platform determinism.
- Keep existing archive semantics unchanged (archive before destructive operations).

Target modules:

- `state.py` (`atomic_write_text`)
- `sync.py` (`stage_skill_dir`, swap/move operations)
- `archive.py` (`archive_move`)

Deliverables:

- Retry helper(s) with jitter/backoff.
- Tests that simulate transient failures via monkeypatch.

### 4) Cross-platform process lifecycle behavior

- Keep Ctrl-C (`SIGINT`) clean exit behavior.
- Guard optional signal registrations for platform differences.
- Document expected stop behavior under:
  - foreground runs (Ctrl-C)
  - Linux `systemd --user`
  - Windows scheduled task stop/termination.

Target module:

- `daemon.py`

Deliverables:

- Safe signal registration with platform checks.
- Updated runtime docs.

### 5) Filename, path, and collision safety

- Extend slug/path safety for Windows:
  - reserved base names (`CON`, `PRN`, `AUX`, `NUL`, `COM1`...`COM9`, `LPT1`...`LPT9`)
  - trailing dot/space normalization issues
  - case-insensitive collision handling.

Target modules:

- `state.py` (`slugify`)
- `sync.py` (collision detection/handling when rendering target names)

Deliverables:

- Deterministic fallback behavior when a slug is invalid/colliding.
- Tests for reserved names and case-only rename collisions.

### 6) Test and CI expansion

- Add Windows jobs to CI matrix (`windows-latest` + existing Linux).
- Keep Linux tests as regression guard.
- Add platform-specific tests for:
  - defaults resolution
  - newline stability
  - transient lock retry behavior
  - reserved filename handling.

Deliverables:

- CI matrix update.
- New and updated tests in `tests/`.

### 7) Documentation update

- Update `README.md`:
  - dual install paths (`install.sh` and `install.ps1`)
  - service management commands for Linux and Windows
  - platform-specific config defaults.
- Update `docs/project_description.md` and constraints language to reflect cross-platform scope.
- Keep `v0.2` history intact; no rewrite of prior phase history.

Deliverables:

- Revised docs with explicit Linux/Windows sections.

## Phased execution plan

1. Platform defaults refactor
- Implement platform-aware defaults and tests.
- No service/installer changes yet.

2. Windows installer and supervision
- Add `install.ps1` + scheduled task registration.
- Add uninstall script.

3. Runtime hardening
- Add atomic write newline control.
- Add retry wrappers for replace/move.

4. Filename/path safety
- Windows reserved names and case-insensitive collision strategy.

5. CI and docs
- Enable Windows CI.
- Update README and project docs.

## Validation plan

### Functional checks

- Claude->Codex and Codex->Claude propagation still passes on Linux and Windows.
- No-op polls remain no-op (no write churn).
- Conflict resolution remains `mtime` based with loser archived.

### Platform checks

- Fresh install works on Windows 10/11 with scheduled background execution.
- Fresh install continues to work on Linux with `systemd --user`.
- Foreground run/stop behavior documented and verified on both platforms.

### Regression checks

- Existing round-trip tests remain green.
- Archive growth remains bounded to destructive-change scenarios.

## Risks and mitigations

- Risk: transient Windows lock errors during rename/move.
  - Mitigation: bounded retries + robust error logging.

- Risk: path default migration surprises for existing users.
  - Mitigation: preserve CLI/config override precedence and document migration clearly.

- Risk: case-insensitive collisions produce ambiguous target paths.
  - Mitigation: deterministic fallback naming and warning logs.

## Done criteria

`v0.3` is complete when all are true:

- Windows install + background run are documented and working.
- Linux install + background run still work unchanged.
- Test suite passes on Linux and Windows CI.
- No regressions in sync correctness, archive safety, or loop suppression behavior.
