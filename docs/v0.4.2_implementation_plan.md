# v0.4.2 implementation plan — US-12 portable library snapshot

This plan implements [US-12](stories/US-12-portable-library-snapshot.md) and the
matching FR-07 / FR-08 from `project_requirements.md`. The work adds two
one-shot CLI subcommands (`export`, `import`) backed by a single new module,
plus a config key for collision strategy. The continuous daemon path is
unchanged.

---

## Phase 0 — State schema bump (v2 → v3): per-pair `last_modified`

US-12 AC-3 / AC-6 require a persisted per-pair "last meaningful change"
timestamp. The existing `state.last_seen` / `last_written` fields are
SHA256 digests, not timestamps, and `mtime` is a per-poll observation
only. We add it.

Changes:

- `STATE_SCHEMA_VERSION = 3` in `src/agents_sync/state.py`.
- `CustomizationArtifactState` gains `last_modified: float | None = None`
  (POSIX seconds). Serialised in `to_dict` / `from_dict`.
- `update_state_n_way` in `src/agents_sync/rendering.py` sets
  `ps.last_modified = time.time()` on every call (every render-and-record
  is a meaningful change). This is the single chokepoint covering
  adoption, sync, conflict resolution, and extend-to-new-tools.
- Migration: existing v2 state files are rebuilt on first boot per the
  existing v0.4 cutover policy (`load_state` returns `{}` on
  schema-mismatch). Re-adoption produces no spurious archive entries
  because the on-disk artifacts already carry their `pair_id`.
- Tests that construct or assert on state shape get a tolerant
  `last_modified` (any positive float allowed; equality compares the rest).

This phase ships as its own commit (`feat(state): bump schema to v3
with per-pair last_modified`).

## Phase 1 — Module: `src/agents_sync/portable_archive.py` (~180 LOC)

Single new module owning all import/export logic. Public surface:

```python
PORTABLE_ARCHIVE_SCHEMA_VERSION = 1

CollisionStrategy = Literal["skip", "mtime_wins", "overwrite"]

@dataclass
class ExportReport:
    artifact_count: int
    archive_path: Path

@dataclass
class ImportReport:
    accepted: list[str]      # pair_ids written
    skipped: list[str]       # pair_ids skipped due to collision policy
    archived_local: list[tuple[str, str]]  # (pair_id, tool) pairs whose tool-side bytes got archived

def export_to_zip(state_dir: Path, zip_path: Path) -> ExportReport: ...
def import_from_zip(
    state_dir: Path,
    zip_path: Path,
    *,
    strategy: CollisionStrategy,
    config: dict[str, Any],
    agentic_tools: dict[str, AgenticToolSpec],
) -> ImportReport: ...
```

**Export internals** (AC-1 to AC-4):

1. `load_state(state_dir)` → in-memory snapshot.
2. For each pair_id: `load_canonical()`, attach
   `last_modified = ps.last_modified` (read from in-memory state),
   hold in a dict. If `last_modified` is None (legacy entry), use 0.0.
3. `tempfile.NamedTemporaryFile` next to `zip_path`, write zip entries from
   memory, `os.replace()` to final path (atomic, AC-4).

**Import internals** (AC-5 to AC-11):

1. `zipfile.ZipFile.open()`; read + validate `manifest.json` (AC-9).
2. Read every `canonical/<id>.json`, validate pair_id (AC-9), parse JSON
   (AC-9), strip `last_modified` from in-memory dict before persisting.
3. `load_state(state_dir)`; build slug index `(kind, slug) → pair_id`.
4. For each imported artifact, classify against existing state: (a) no
   collision; (b) pair_id collision; (c) slug collision with different
   pair_id. Apply `strategy` → list of `accepted` and `skipped`.
5. **Decision phase ends before any disk write.** This gives AC-10's
   "transactional state.json" guarantee.
6. For each accepted artifact:
   - If displacing a local entry (mtime_wins where import wins, or
     overwrite): `archive.archive_copy` every existing tool-side file
     recorded in local state (NFR-01 → AC-6, AC-7).
   - `save_canonical(state_dir, pair_id, canonical_minus_last_modified)`.
   - Build new `CustomizationArtifactState` with `last_written=None`,
     `last_seen=None`, `path` = computed local target path for each
     enabled+supporting tool, using `tool_root(...)` + `target_slug(name)`
     + the IO module's file-layout descriptor.
7. `save_state(state_dir, mutated_state)` last (atomic — existing helper).

## Phase 2 — Config: `src/agents_sync/config.py`

Add to `platform_defaults()`:

```python
"import_collision_strategy": "mtime_wins",
```

Add to `validate_config()`:

```python
strategy = config.get("import_collision_strategy", "mtime_wins")
if strategy not in {"skip", "mtime_wins", "overwrite"}:
    raise ConfigError(
        f"import_collision_strategy must be skip|mtime_wins|overwrite, got {strategy!r}"
    )
```

## Phase 3 — CLI: `src/agents_sync/cli.py`

Convert to subparsers while keeping the no-subcommand form running the
daemon (back-compat):

```python
sub = parser.add_subparsers(dest="command")

export_p = sub.add_parser("export", help="Write a portable library snapshot to a zip file.")
export_p.add_argument("output", type=Path)

import_p = sub.add_parser("import", help="Restore a portable library snapshot from a zip file.")
import_p.add_argument("input", type=Path)
import_p.add_argument(
    "--collision-strategy",
    choices=["skip", "mtime_wins", "overwrite"],
    default=None,  # falls back to config
)
```

In `main()`:

- `args.command == "export"` → call `export_to_zip`, log report, return 0/1.
- `args.command == "import"` → resolve strategy (CLI flag wins over
  config), call `import_from_zip`, log report, return 0/1.
- `args.command is None` → existing daemon path (unchanged).

## Risks / things to flag if they surface

- **`last_modified` is wall-clock and not guaranteed monotonic across
  hosts.** If someone exports from a host with a skewed clock,
  `mtime_wins` can be misleading. The module docstring documents the
  caveat; the AC-6 tiebreak rule (local wins on tie) is the
  default-deny fallback.
- **Slug collision detection** uses the imported canonical's `name`
  field directly — no per-tool parse needed during import, keeping the
  import path tool-agnostic.
- **Windows zip path separators.** `zipfile` uses forward slashes
  regardless of OS; entries are `canonical/<id>.json`. A round-trip test
  on Windows-style state paths catches any leak.
